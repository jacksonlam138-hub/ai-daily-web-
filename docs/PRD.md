## 1. 产品概述

### 1.1 产品定位

**AI日报** 是一款面向AI从业者、产品经理、开发者的自动化信息聚合SaaS产品，通过多源数据聚合和AI智能处理，每日清晨自动生成结构化的AI行业日报并推送到用户邮箱。

### 1.2 目标用户

| 用户类型 | 描述 | 核心诉求 |
|---------|------|---------|
| AI从业者 | 算法工程师、数据科学家 | 追踪学术前沿、技术动态 |
| 产品经理 | AI产品相关PM | 了解竞品动态、市场趋势 |
| 开发者 | 应用层开发者 | 获取API更新、工具发布 |
| 投资人/分析师 | 关注AI行业投资机会 | 融资信息、商业动态 |

### 1.3 核心痛点

| 痛点 | 描述 | 现有解决方案缺陷 |
|-----|------|----------------|
| 来源单一 | 信息渠道有限，无法全面覆盖 | 人工收集效率低 |
| 无链接 | 缺乏原文链接，无法深入了解 | 无法追溯信息源 |
| 竞品动态不足 | 竞品信息分散，更新不及时 | 缺乏系统化监控 |
| 缺趋势分析 | 信息碎片化，缺乏深度洞察 | 无自动化分析能力 |
| 格式混乱 | 不同来源格式不统一 | 阅读体验差 |

### 1.4 产品价值

- **效率提升**: 节省用户每日30-60分钟信息收集时间
- **信息全面**: 10+优质AI信息源聚合
- **格式统一**: 标准化5板块结构，阅读体验佳
- **可追溯**: 100%来源链接覆盖
- **个性化**: 支持领域偏好定制

## 2. 用户故事

### 2.1 核心用户故事

#### US-001: 多源信息聚合
```
作为一名 AI从业者
我希望 系统能自动聚合多个AI信息源
以便于 我能在一个地方获取全面的信息，节省收集时间

验收标准:
- [ ] 系统支持至少10个AI信息源
- [ ] 每日自动抓取更新
- [ ] 抓取成功率 > 95%
```

#### US-002: 标准化日报生成
```
作为一名 产品经理
我希望 每天收到格式统一的AI日报
以便于 我能快速浏览和定位感兴趣的内容

验收标准:
- [ ] 日报包含5个固定板块
- [ ] 每个板块有清晰的标题和分类
- [ ] 格式美观，易于阅读
```

#### US-003: 来源链接追溯
```
作为一名 开发者
我希望 每条信息都包含原文链接
以便于 我能深入了解感兴趣的资讯

验收标准:
- [ ] 每条信息包含可点击的来源链接
- [ ] 链接覆盖率 100%
- [ ] 链接有效率 > 98%
```

#### US-004: 邮件自动推送
```
作为一名 AI从业者
我希望 每天早上自动收到日报邮件
以便于 我在通勤途中就能了解行业动态

验收标准:
- [ ] 支持邮件订阅
- [ ] 每日固定时间推送（默认早8点）
- [ ] 支持自定义推送时间
```

#### US-005: 领域定制
```
作为一名 专注NLP的算法工程师
我希望 只接收与我领域相关的内容
以便于 我能聚焦关注点，减少信息噪音

验收标准:
- [ ] 支持选择感兴趣的领域（CV/NLP/RL等）
- [ ] 定制后日报优先展示相关内容
- [ ] 支持关键词过滤
```

### 2.2 用户故事优先级矩阵

| ID | 用户故事 | 优先级 | 迭代 |
|----|---------|-------|-----|
| US-001 | 多源信息聚合 | P0 | MVP |
| US-002 | 标准化日报生成 | P0 | MVP |
| US-003 | 来源链接追溯 | P0 | MVP |
| US-004 | 邮件自动推送 | P0 | MVP |
| US-005 | 领域定制 | P1 | MVP |

## 3. 功能需求

### 3.1 多源聚合模块

#### 3.1.1 信息源管理

**功能描述**: 管理和维护AI信息源列表

| 字段 | 类型 | 说明 |
|-----|------|-----|
| source_id | string | 信息源唯一标识 |
| name | string | 信息源名称 |
| url | string | 信息源地址 |
| type | enum | 类型: rss/api/web |
| category | array | 分类标签 |
| crawl_freq | string | 抓取频率 |
| status | enum | 状态: active/inactive |

**验收标准**:
- AC-3.1.1-1: 支持添加/编辑/删除信息源
- AC-3.1.1-2: 支持RSS、API、网页三种类型
- AC-3.1.1-3: 支持启用/禁用信息源
- AC-3.1.1-4: 系统预置10+高质量AI信息源

**预置信息源列表（MVP）**:

| 序号 | 信息源 | 类型 | 分类 |
|-----|-------|------|-----|
| 1 | arXiv cs.AI/cs.CL/cs.CV | API | 学术 |
| 2 | OpenAI Blog | RSS | 产品 |
| 3 | Google AI Blog | RSS | 学术/产品 |
| 4 | Meta AI Blog | RSS | 学术/产品 |
| 5 | Anthropic Blog | RSS | 产品 |
| 6 | The Verge AI | RSS | 商业 |
| 7 | TechCrunch AI | RSS | 商业 |
| 8 | Hugging Face Blog | RSS | 技术 |
| 9 | MIT Technology Review | RSS | 趋势 |
| 10 | VentureBeat AI | RSS | 融资 |

#### 3.1.2 内容抓取

**功能描述**: 定时从各信息源抓取最新内容

**验收标准**:
- AC-3.1.2-1: 每日至少抓取1次
- AC-3.1.2-2: 抓取成功率 > 95%
- AC-3.1.2-3: 支持增量抓取，避免重复
- AC-3.1.2-4: 支持错误重试机制（3次）
- AC-3.1.2-5: 记录抓取日志，便于排查

**技术要点**:
- 使用OpenClaw后端进行爬虫抓取
- 遵守robots.txt协议
- 设置合理的请求频率，避免被封禁

#### 3.1.3 内容去重

**功能描述**: 对抓取的内容进行去重处理

**验收标准**:
- AC-3.1.3-1: 基于标题相似度去重（阈值>0.85）
- AC-3.1.3-2: 基于URL去重
- AC-3.1.3-3: 保留最新、最权威来源

### 3.2 日报生成模块

#### 3.2.1 内容分类

**功能描述**: 将抓取的内容自动分类到5个板块

**板块结构**:

| 板块 | 名称 | 内容范围 |
|-----|------|---------|
| 1 | 今日要点 | 重大新闻、热点事件 |
| 2 | 学术前沿 | 论文发布、研究成果 |
| 3 | 产品发布 | 新产品、功能更新 |
| 4 | 融资&商业 | 融资、收购、合作 |
| 5 | 数据速览 | 关键数据、报告摘要 |

**验收标准**:
- AC-3.2.1-1: 使用AI自动分类，准确率 > 90%
- AC-3.2.1-2: 支持人工审核调整
- AC-3.2.1-3: 每个板块最多展示10条

#### 3.2.2 内容摘要

**功能描述**: 使用AI生成每条内容的精炼摘要

**验收标准**:
- AC-3.2.2-1: 摘要长度控制在50-100字
- AC-3.2.2-2: 摘要准确反映原文核心内容
- AC-3.2.2-3: 控制AI调用成本（单篇<0.01元）

#### 3.2.3 日报模板

**功能描述**: 定义日报的输出格式

**模板结构**:
```markdown
# AI日报 | {YYYY-MM-DD}

> 每日精选AI行业资讯，助您把握前沿动态

---

## 今日要点

### 1. {标题}
{摘要}
[原文链接]({url}) | 来源: {source}

---

## 学术前沿

### 1. {标题}
{摘要}
[原文链接]({url}) | 来源: {source}

---

## 产品发布
...

## 融资&商业
...

## 数据速览
...

---

*本日报由AI自动生成，如有问题请反馈*
*退订 | 设置 | 反馈*
```

**验收标准**:
- AC-3.2.3-1: 格式美观，支持邮件客户端兼容
- AC-3.2.3-2: 包含退订链接
- AC-3.2.3-3: 响应式设计，支持移动端阅读

### 3.3 邮件推送模块

#### 3.3.1 订阅管理

**功能描述**: 用户订阅/退订管理

**验收标准**:
- AC-3.3.1-1: 支持邮箱订阅
- AC-3.3.1-2: 发送确认邮件验证邮箱
- AC-3.3.1-3: 支持一键退订
- AC-3.3.1-4: 记录订阅状态和时间

#### 3.3.2 推送调度

**功能描述**: 定时推送日报邮件

**验收标准**:
- AC-3.3.2-1: 默认推送时间 08:00（用户时区）
- AC-3.3.2-2: 支持自定义推送时间
- AC-3.3.2-3: 推送成功率 > 99%
- AC-3.3.2-4: 记录推送状态和日志

#### 3.3.3 邮件模板

**功能描述**: 邮件HTML模板

**验收标准**:
- AC-3.3.3-1: 兼容主流邮件客户端（Gmail, Outlook, QQ邮箱等）
- AC-3.3.3-2: 支持深色/浅色模式
- AC-3.3.3-3: 包含品牌元素

### 3.4 用户定制模块

#### 3.4.1 领域选择

**功能描述**: 用户选择关注的领域

**可选领域**:
- 计算机视觉 (CV)
- 自然语言处理 (NLP)
- 强化学习 (RL)
- 多模态 (Multimodal)
- 生成式AI (Generative AI)
- AI基础设施 (Infrastructure)
- AI应用 (Applications)

**验收标准**:
- AC-3.4.1-1: 支持多选
- AC-3.4.1-2: 选择后优先展示相关内容
- AC-3.4.1-3: 未选择时展示全量内容

#### 3.4.2 关键词过滤

**功能描述**: 基于关键词的内容过滤

**验收标准**:
- AC-3.4.2-1: 支持添加关注关键词
- AC-3.4.2-2: 支持添加屏蔽关键词
- AC-3.4.2-3: 关键词支持中英文
- AC-3.4.2-4: 每个用户最多设置20个关键词

## 6. 接口需求

### 6.1 OpenClaw后端接口对接

#### 6.1.1 爬虫任务接口

**接口**: `POST /api/v1/crawl/tasks`

**请求体**:
```json
{
  "source_id": "string",
  "source_url": "string",
  "source_type": "rss|api|web",
  "crawl_config": {
    "frequency": "daily",
    "selectors": {}
  }
}
```

**响应体**:
```json
{
  "task_id": "string",
  "status": "pending|running|completed|failed",
  "articles_count": 0,
  "created_at": "timestamp"
}
```

#### 6.1.2 爬虫结果接口

**接口**: `GET /api/v1/crawl/results/{task_id}`

**响应体**:
```json
{
  "task_id": "string",
  "status": "completed",
  "articles": [
    {
      "title": "string",
      "content": "string",
      "url": "string",
      "published_at": "timestamp"
    }
  ]
}
```

#### 6.1.3 信息源管理接口

| 接口 | 方法 | 说明 |
|-----|------|-----|
| `/api/v1/sources` | GET | 获取信息源列表 |
| `/api/v1/sources` | POST | 添加信息源 |
| `/api/v1/sources/{id}` | PUT | 更新信息源 |
| `/api/v1/sources/{id}` | DELETE | 删除信息源 |

### 6.2 AI服务接口

#### 6.2.1 内容分类接口

**接口**: `POST /api/v1/ai/classify`

**请求体**:
```json
{
  "articles": [
    {
      "id": "string",
      "title": "string",
      "content": "string"
    }
  ]
}
```

**响应体**:
```json
{
  "results": [
    {
      "article_id": "string",
      "category": "breaking|academic|product|business|data",
      "confidence": 0.95
    }
  ]
}
```

#### 6.2.2 摘要生成接口

**接口**: `POST /api/v1/ai/summarize`

**请求体**:
```json
{
  "articles": [
    {
      "id": "string",
      "title": "string",
      "content": "string",
      "max_length": 100
    }
  ]
}
```

**响应体**:
```json
{
  "results": [
    {
      "article_id": "string",
      "summary": "string",
      "token_count": 0
    }
  ]
}
```

### 6.3 邮件服务接口

**接口**: `POST /api/v1/email/send`

**请求体**:
```json
{
  "to": "string",
  "subject": "string",
  "html_body": "string",
  "text_body": "string"
}
```

### 6.4 用户服务接口

| 接口 | 方法 | 说明 |
|-----|------|-----|
| `/api/v1/users/subscribe` | POST | 用户订阅 |
| `/api/v1/users/unsubscribe` | POST | 用户退订 |
| `/api/v1/users/preferences` | GET/PUT | 获取/更新用户偏好 |
